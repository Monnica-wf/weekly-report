# 周报系统 - 时间线功能使用指南

## 功能概述

本次更新为周报系统添加了时间线功能，支持在工作项中添加里程碑节点，帮助追踪工作进度。

---

## ✅ 功能 1：编辑本周工作内容

**如何使用：**

1. 在"本周工作"区域，每个工作项卡片右上角有两个按钮
2. 点击 ✏️ **铅笔图标**（编辑按钮）
3. 在弹出的编辑框中修改所有字段：
   - 标题
   - 详细内容
   - 分类（AI / C端产品）
   - 优先级（P0-P3）
   - 标签
   - 开始/完成日期
   - 截止日期（DDL）
   - 颜色标记
   - 完成进度
   - **里程碑节点（新增）**

---

## ✅ 功能 2：时间线模块

**位置：** 
在页面顶部 Hero 概览区域下方，新增了"本周时间线"模块

**功能说明：**
- 自动聚合显示所有工作项的里程碑节点
- 按时间排序展示
- 实时显示里程碑总数

**节点颜色说明：**

| 颜色 | 状态 | 说明 |
|------|------|------|
| 🟢 绿色 | 已完成 | 里程碑已完成 |
| 🟡 黄色 | 进行中 | 正在处理中 |
| ⚪ 灰色 | 待完成 | 等待开始 |
| 🔴 红色 | 已逾期 | 超过预期日期但未完成 |

---

## ✅ 功能 3：里程碑管理

### 如何添加里程碑

1. 点击"添加工作"或编辑现有工作项
2. 在编辑弹窗底部找到 **"里程碑节点"** 区域
3. 点击 **"添加里程碑"** 按钮
4. 填写里程碑信息：
   - **名称**：如"需求评审"、"开发完成"、"测试中"、"验收上线"
   - **预期日期**：计划完成时间
   - **状态**：待完成 / 进行中 / 已完成

### 如何编辑里程碑

- 直接在输入框中修改名称
- 点击日期选择器修改预期日期
- 通过下拉菜单修改状态

### 如何删除里程碑

- 点击里程碑右侧的 🗑️ 删除按钮

## 使用示例

### 场景：安全工单进线链路优化

**步骤：**

1. 添加新工作项：点击"添加工作"
2. 填写基本信息：
   - 标题：安全工单进线链路优化
   - 进度：50%
   - DDL：2026-06-30

3. 添加里程碑：
   - ✅ 需求评审 - 06/20 - 已完成
   - ✅ 开发完成 - 06/22 - 已完成
   - 🟡 测试中 - 06/25 - 进行中
   - ⏳ 验收上线 - 06/28 - 待完成

4. 保存后，时间线自动显示：
   ```
   安全工单进线链路优化
   ● 06/20 需求评审 ✅
   ● 06/22 开发完成 ✅
   🟡 06/25 测试中
   ○ 06/28 验收上线
   ```

## 技术细节

### 数据结构

每个工作项新增 `milestones` 字段：

```javascript
{
  id: 123456,
  title: "安全工单进线链路优化",
  milestones: [
    {
      id: 789,
      title: "需求评审",
      expectedDate: "2026-06-20",
      status: "completed"  // completed | in_progress | pending
    }
  ]
}
```
